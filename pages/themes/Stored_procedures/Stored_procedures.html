User Defined Variables

User Defined Variables

SET variable

A User Defined Variable is written as @var_name where, var_name is the name of the variable and can consist of alphanumeric characters, '.', '_', and '$'.
Variable name can include other characters like- {!, #, ^, -, ..} if they are quoted
User Defined Variable is defined with SET statement:

				SET @var_name = expr [, @var_name = expr]
Example:

				SET @myName = 'Iva E. Popova';

				SELECT @myName;
				+---------------+
				| @myName       |
				+---------------+
				| Iva E. Popova |
				+---------------+
Example 2

				SET @a=3,@b=4;
				SET @rect_area=@a*@b;

				SELECT @a,@b,@rect_area;
				+----+----+------------+
				| @a | @b | @rect_area |
				+----+----+------------+
				| 3  | 4  | 12         |
				+----+----+------------+

Assign variable in SELECT

We can assign a value to a variable in the SELECT statement using the := operator.

				SELECT @min_price:=MIN(price) FROM books;
We can also use the SELECT ... INTO Statement

				SELECT MAX(price) INTO @max_price FROM books;

Use Case

User Defined Variables enables you to pass values from one statement to another.
You can store a value in a user-defined variable in one statement and refer to it later in another statement.

				SELECT @min_price:=MIN(price), @max_price:=MAX(price)
				FROM books;

				SELECT * FROM books
				WHERE price IN (@min_price, @max_price);

				+----+-----------+----------------------------+----------+--------+
				| id | author_id | book_name                  | pub_year | price  |
				+----+-----------+----------------------------+----------+--------+
				| 11 | 2         | Young Zaphod Plays It Safe | 1986     | 5.60   |
				| 35 | 3         | Curiosa Mathematica I      | 1888     | 295.40 |
				+----+-----------+----------------------------+----------+--------+

Notes

User Defined Variable can take values only from following data types: integer, floating-point, decimal, binary, non-binary string or NULL
A value of a type other than one of the permissible types is converted to a permissible type
Assignment of decimal and real values does not preserve the precision or scale of the value
User-defined variables are session specific
A user-defined variable set by one client can not be seen by other client
All variables for a given client session are automatically freed when that client exits.
Reference: User-Defined Variables

Server System Variables

Server System Variables

The MySQL server maintains many system variables configured to a default value
System Variables are prefixed with '@@'
To see the current values of a system variable use the SHOW VARIABLES statement or SELECT @@var_name.

			SHOW VARIABLES LIKE 'connect_timeout';
			+-----------------+-------+
			| Variable_name   | Value |
			+-----------------+-------+
			| connect_timeout | 10    |
			+-----------------+-------+

			SELECT @@connect_timeout;
			+-------------------+
			| @@connect_timeout |
			+-------------------+
			| 10                |
			+-------------------+

Stored Procedures (SP)

Stored Procedures (SP)

Overview

A Stored Procedure in SQL is a group of pre-compiled SQL statements that are stored together under unique name in a database.
when you drop a database, any stored routines () in the database are also dropped.
SP allows for quick reuse of language constructs
thus, reduce the amount of code that must be written
Better Performance
Stored Procedures are compiled once and stored in executable form, thus the response is quick.
The executable code is automatically cached, thus lowers the memory and network traffic.

Create and Call a Procedure

Create Procedure (Reference: CREATE PROCEDURE )

				[DELIMITER {custom delimiter}]
				CREATE PROCEDURE sp_name ([proc_parameter[,...]])
				BEGIN
				-- execution code
				END;
				[DELIMITER ;] -- reset custom delimiter
It’s recommended to have the stored proc names begin with sp_ or sp to identify that it’s a proc.
Each proc_parameter can be of the form: [IN | OUT | INOUT] param_name param_type
  • IN param: allows you to pass values into the procedure
  • OUT param: allows you to pass value back from procedure to the calling program
By default, mysql itself recognizes the semicolon as a statement delimiter, so you must redefine the delimiter temporarily to cause mysql to pass the entire stored program definition to the server.
Example

				CREATE PROCEDURE spUpdatePrice(IN id_to_update int, IN new_price decimal)
				BEGIN
					UPDATE books SET price=new_price where id=id_to_update;
				END;
Execute a Procedure

				CALL spUpdatePrice(1,999);

Delete Stored Procedure

The DROP PROCEDURE statement deletes a stored procedure created by the CREATE PROCEDURE statement.

				DROP PROCEDURE [IF EXISTS] stored_procedure_name;
			

				DROP PROCEDURE spUpdatePrice;

Example: SP with IN parameters

Create the spGetBooksByAuthorFname procedure

				DELIMITER //
				CREATE PROCEDURE spGetBooksByAuthorFname(IN author_fname VARCHAR(200))

					SELECT * FROM books as b
					JOIN authors as a
					ON b.author_id=a.id
					WHERE a.fname=author_fname;
				END;
				DELIMITER ;
Use spGetBooksByAuthorFname

Example: SP with IN and OUT parameters

Create the spGetMostExpensiveAuthorBook procedure

				DELIMITER //
				CREATE PROCEDURE spGetMostExpensiveAuthorBook
				(IN author_fname VARCHAR(200), OUT max_price DECIMAL(6.2))
				BEGIN
					SELECT MAX(b.price) INTO max_price
					FROM books as b
					JOIN authors as a
					ON b.author_id=a.id
					WHERE a.fname=author_fname;
				END;
				DELIMITER ;
Use spGetMostExpensiveAuthorBook
As we have an OUT parameter that would be returning a value, we must specify a variable as a placeholder to retrieve the returned value

				CALL spGetMostExpensiveAuthorBook('Kurt', @max_book_price);
				-- Query OK, 1 row affected

				SELECT @max_book_price;
				+-----------------+
				| @max_book_price |
				+-----------------+
				| 999             |
				+-----------------+

Example: SP with INOUT parameters

An INOUT parameter is initialized by the caller, can be modified by the procedure, and any change made by the procedure is visible to the caller when the procedure returns.
Create the spIncrementCount procedure

				DELIMITER //
				CREATE PROCEDURE spIncrementCount
				(INOUT counter INT, IN increment INT)
				BEGIN
					SET counter=counter+increment;
				END;
				DELIMITER ;
Use spIncrementCount

				-- set initial @counter value:
				SET @counter = 1;

				-- increment @counter with 5:
				CALL spIncrementCount(@counter,5);

				-- check the result:
				SELECT @counter;
				+----------+
				| @counter |
				+----------+
				| 6        |
				+----------+

Stored Functions (SF)

Stored Functions (SF)

Overview

Stored function is a special kind stored program that MUST returns a single value
With SF we can increase the expressiveness of MySQL as we can use our functions to encapsulate common formulas or business rules that are reusable among SQL statements
We can use a stored function in SQL statements wherever an expression is used, just like the built-in MySQL functions

Create Stored Function

Basic Syntax (Reference: CREATE PROCEDURE and CREATE FUNCTION Statements )

				DELIMITER //

				CREATE FUNCTION function_name([func_parameter[,...]])
				RETURNS datatype
				[[NOT] DETERMINISTIC]
				BEGIN
				-- statements (must have RETURN expr)
				END //

				DELIMITER ;
By default, all func_parameters are the IN parameters. You cannot specify IN , OUT or INOUT modifiers to parameters
We must specify the data type of the return value in the RETURNS statement, which can be any valid MySQL data types.
If we don’t specify DETERMINISTIC, MySQL uses the NOT DETERMINISTIC option by default.
A deterministic function always returns the same result for the same input parameters whereas a non-deterministic function returns different results for the same input parameters.
Inside the body (BEGIN...END) section, we mist specify at least one RETURN statement
The RETURN statement returns a value to the calling programs
Whenever the RETURN statement is reached, the execution of the stored function is terminated immediately

Example: incrementCount()

Create incrementCount function

				CREATE FUNCTION incrementCount
				(counter INT, increment INT)
				RETURNS INT
				DETERMINISTIC
				BEGIN
					RETURN counter+increment;
				END;
We can use incrementCount function directly in select:

				SELECT incrementCount(1,5);
Or store the return value in custom variable

				SET @counter = 1;

				SET @counter=incrementCount(@counter,5);

				SELECT @counter;

				+----------+
				| @counter |
				+----------+
				| 6        |
				+----------+

Example: levToDollar()

Create levToDollar function

				CREATE FUNCTION levToDollar
				(lev DECIMAL(10,2) UNSIGNED)
				RETURNS DECIMAL(10,2) UNSIGNED
				DETERMINISTIC
				BEGIN
					RETURN lev*0.54;
				END;
Use levToDollar

				SELECT levToDollar(price) as price_in_dollars
				FROM books
				LIMIT 5;

				+------------------+
				| price_in_dollars |
				+------------------+
				| 539.46           |
				| 7.70             |
				| 10.81            |
				| 12.69            |
				| 24.30            |
				+------------------+

Local Variables in Stored Routines

Local Variables in Stored Routines

Overview

We can use Local Variables inside Stored Routines for temporary storage of data concerning the routine.
Stored Routines refers to both Stored Procedures and Stored Functions
Local variables have a scope limited to the routine itself.
Local variables needs to be declared using DECLARE before accessing it.

				DECLARE varName DATATYPE [DEFAULT value];
DECLARE is permitted only inside a BEGIN ... END compound statement and must be at its start, before any other statements.

				-- if we write that outside stored program, an Error will be raised
				DECLARE x INT;
No @ prefix like user-defined variable is needed when declaring Local variable.
Local variables can be used as input parameters inside a stored procedure, as well.

Example


			DELIMITER //
			CREATE FUNCTION spGetBookPriceLevel(book_id INT)
			RETURNS VARCHAR(20)
			DETERMINISTIC
			BEGIN
						DECLARE book_level VARCHAR(20) DEFAULT 'none';
				DECLARE book_price DECIMAL(5,2) DEFAULT 0;

				--Get book price:
				SELECT price INTO book_price
				FROM books
				WHERE books.id=book_id;

				-- Define book level
				CASE
					WHEN book_price<=10 THEN
						SET book_level='Cheep';
					WHEN (book_price>10 AND book_price<50) THEN
						SET book_level='Affordable';
					WHEN book_price>=50 THEN
						SET book_level='Expensive';
				END CASE;

				-- Return the book level
				RETURN book_level;
			END //

			DELIMITER ;

Stored Procedures vs Stored Functions

Stored Procedures vs Stored Functions

Stored Procedure Stored Functions
Can return zero, single or multiple values. Must return exactly one value.
SP can have input/output parameters SF can have only input parameters
SP can not be used inside SELECT, WHERE or HAVING clauses. SF can be used inside SELECT, WHERE or HAVING clauses.
A procedure is invoked with CALL statement to perform an operation such as modifying a table or processing retrieved records. A function is invoked within an expression and returns a single value directly to the caller to be used in the expression.
