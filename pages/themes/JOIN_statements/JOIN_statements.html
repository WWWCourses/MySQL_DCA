MySQL INNER JOIN clause

MySQL INNER JOIN clause

Overview

In MySQL, JOIN and INNER JOIN are syntactic equivalents (they can replace each other).
Basic syntax:

				SELECT column_list
				FROM table_1
				INNER JOIN table_2 ON join_condition;
The inner join clause joins two tables based on a condition.
The inner join clause compares each row from the first table with every row from the second table.If values from both rows satisfy the join condition, the inner join clause creates a new row whose column contains all columns of the two rows from both tables and includes this new row in the result set.
In other words, the inner join clause includes only matching rows from both tables.

Example


			SELECT *
			FROM users AS u INNER JOIN addresses AS a
			ON u.id = a.user_id;
		

			+----+-------+----+---------+-------+
			| id | name  | id | user_id | city  |
			+----+-------+----+---------+-------+
			| 1  | Ivan  | 1  | 1       | Sofia |
			| 3  | Merry | 2  | 3       | Paris |
			+----+-------+----+---------+-------+

MySQL LEFT [OUTER] JOIN

Overview

Basic Syntax:

				SELECT column_list
				FROM table_1
				LEFT JOIN table_2 ON join_condition;
LEFT JOIN selects all data from the left table whether there are matching rows exist in the right table or not
In case there are no matching rows from the right table found, the left join uses NULL values for columns of the row from the right table in the result set.

Example


			SELECT *
			FROM users AS u LEFT JOIN addresses AS a
			ON u.id = a.user_id;
		

			+----+-------+--------+---------+--------+
			| id | name  | id     | user_id | city   |
			+----+-------+--------+---------+--------+
			| 1  | Ivan  | 1      | 1       | Sofia  |
			| 2  | John  | <null> | <null>  | <null> |
			| 3  | Merry | 2      | 3       | Paris  |
			| 4  | Pesho | <null> | <null>  | <null> |
			+----+-------+--------+---------+--------+

LEFT JOIN with exclusion

It gives all records of the left table (Table1) excluding the common elements from right table(Table2)

LEFT JOIN with exclusion - example


			SELECT *
			FROM users AS u LEFT JOIN addresses AS a
			ON u.id = a.user_id
			WHERE a.user_id IS NULL;
		

			+----+-------+--------+---------+--------+
			| id | name  | id     | user_id | city   |
			+----+-------+--------+---------+--------+
			| 2  | John  | <null> | <null>  | <null> |
			| 4  | Pesho | <null> | <null>  | <null> |
			+----+-------+--------+---------+--------+

MySQL RIGHT [OUTER] JOIN

Overview

Basic Syntax:

				SELECT column_list
				FROM table_1
				RIGHT JOIN table_2 ON join_condition;
The right join clause is similar to the left join clause except that the treatment of left and right tables is reversed.
The right join clause selects all rows from the right table and matches rows in the left table. If a row from the right table does not have matching rows from the left table, the column of the left table will have NULL in the final result set.

Example


			SELECT u.name, a.city
			FROM users AS u RIGHT JOIN addresses AS a
			ON u.id = a.user_id;
		

			+--------+----------+
			| name   | city     |
			+--------+----------+
			| Ivan   | Sofia    |
			| Merry  | Paris    |
			| <null> | London   |
			| <null> | New York |
			+--------+----------+

RIGHT JOIN with exclusion

It gives all records of the right table (Table2) excluding the common elements from left table(Table1)

RIGHT JOIN with exclusion - example


			SELECT *
			FROM users AS u RIGHT JOIN addresses AS a
			ON u.id = a.user_id
			WHERE u.id IS NULL;
		

			+--------+--------+----+---------+----------+
			| id     | name   | id | user_id | city     |
			+--------+--------+----+---------+----------+
			| <null> | <null> | 3  | <null>  | London   |
			| <null> | <null> | 4  | <null>  | New York |
			+--------+--------+----+---------+----------+

MySQL FULL OUTER JOIN

Overview

Note that MySQL did not support FULL OUTER JOIN, but we can emulate it with: LEFT JOIN UNION [ALL] RIGHT JOIN

				SELECT column_list
				FROM table_1
				LEFT JOIN table_2 ON join_condition
				UNION
				SELECT column_list
				FROM table_1
				RIGHT JOIN table_2 ON join_condition
The UNION clause is used to combine the result-set of two or more SELECT statements. IT selects only distinct values by default. To allow duplicate values we can use UNION ALL
FULL OUTER JOIN returns all records when there is a match in left (table1) or right (table2) table records

Example


			SELECT *
			FROM users AS u LEFT JOIN addresses AS a
			ON u.id = a.user_id

			UNION

			SELECT *
			FROM users AS u RIGHT JOIN addresses AS a
			ON u.id = a.user_id;
		

			+--------+--------+--------+---------+----------+
			| id     | name   | id     | user_id | city     |
			+--------+--------+--------+---------+----------+
			| 1      | Ivan   | 1      | 1       | Sofia    |
			| 2      | John   | <null> | <null>  | <null>   |
			| 3      | Merry  | 2      | 3       | Paris    |
			| 4      | Pesho  | <null> | <null>  | <null>   |
			| <null> | <null> | 3      | <null>  | London   |
			| <null> | <null> | 4      | <null>  | New York |
			+--------+--------+--------+---------+----------+

FULL OUTER JOIN with exclusion

It gives all uncommon records of both the tables

FULL OUTER JOIN with exclusion - example


			SELECT *

			ON u.id = a.user_id
			WHERE a.user_id IS  NULL

			UNION

			SELECT *
			FROM users AS u RIGHT JOIN addresses AS a
			ON u.id = a.user_id

			WHERE u.id IS NULL;
		

			+--------+--------+--------+---------+----------+
			| id     | name   | id     | user_id | city     |

			| 4      | Pesho  | <null> | <null>  | <null>   |
			| <null> | <null> | 3      | <null>  | London   |
			| <null> | <null> | 4      | <null>  | New York |

MySQL SEMI JOIN

Overview

Note that MySQL did not support SEMI JOIN, but we can emulate it with EXISTS

				SELECT column_list
				FROM table_1
				WHERE EXISTS (
					SELECT * FROM table_2
					WHERE join_condition
				);
SEMI JOIN returns more restricted set that INNER JOIN.
With SEMI JOIN, we get only the first matching record in the left hand side table in the output. As soon as a match is hit, the join will move to the next id.
SEMI JOIN returns only columns from table_1.

Example

Let's add multiple records in child table to see the difference between INNER JOIN and SEMI JOIN

				UPDATE addresses SET user_id=1 WHERE addresses.id=3;

				SELECT * FROM addresses;
				+----+---------+----------+
				| id | user_id | city     |
				+----+---------+----------+
				| 1  | 1       | Sofia    |
				| 2  | 3       | Paris    |
				| 3  | 1       | London   |
				| 4  | <null>  | New York |
				+----+---------+----------+
Now lets perform SEMI JOIN and INNER JOIN:

					SELECT *
					FROM users AS u
					WHERE EXISTS (
						SELECT * FROM addresses AS a
						WHERE u.id = a.user_id
					);

					+----+-------+
					| id | name  |
					+----+-------+
					| 1  | Ivan  |
					| 3  | Merry |
					+----+-------+
				

					SELECT *
					FROM users as u INNER JOIN addresses as a
					ON u.id = a.user_id;

					+----+-------+----+---------+-------+
					| id | name  | id | user_id | city  |
					+----+-------+----+---------+-------+
					| 1  | Ivan  | 1  | 1       | Sofia |
					| 3  | Merry | 2  | 3       | Paris |
					| 1  | Ivan  | 3  | 1       | London|
					+----+-------+----+---------+-------+

MySQL ANTI SEMI JOIN

Overview

Note that MySQL did not support ANTI SEMI JOIN, but we can emulate it with NOT EXISTS:

				SELECT column_list
				FROM table_1
				WHERE NOT EXISTS (
					SELECT * FROM table_2
					WHERE join_condition
				);
With ANTI SEMI JOIN we get the rows from table_1 which did not have any matching records in table_2

Example


			SELECT *
			FROM users AS u
			WHERE NOT EXISTS (
				SELECT * FROM addresses AS a
				WHERE u.id = a.user_id
			);
		

			+----+-------+
			| id | name  |
			+----+-------+
			| 2  | John  |
			| 4  | Pesho |
			+----+-------+

MySQL CROSS JOIN

Overview

Basic syntax

				-- with explicit syntax:
				SELECT *
				FROM table_1
				CROSS JOIN table2;

				-- next is the same, but implicit:
				SELECT * from table_1, table_2;
The SQL Server Cross Join returns the Cartesian product of both tables.
It means every record in table_1 is combined with every record in table_2. The resulting records number is equal to the multiplication of table_1 and table2 records, and can become very huge!
The CROSS JOIN is used to generate all combinations of records in two tables

Example


			SELECT *
			FROM users
			CROSS JOIN addresses;
		

			+----+-------+----+---------+----------+
			| id | name  | id | user_id | city     |
			+----+-------+----+---------+----------+
			| 4  | Pesho | 1  | 1       | Sofia    |
			| 3  | Merry | 1  | 1       | Sofia    |
			| 2  | John  | 1  | 1       | Sofia    |
			| 1  | Ivan  | 1  | 1       | Sofia    |
			| 4  | Pesho | 2  | 3       | Paris    |
			| 3  | Merry | 2  | 3       | Paris    |
			| 2  | John  | 2  | 3       | Paris    |
			| 1  | Ivan  | 2  | 3       | Paris    |
			| 4  | Pesho | 3  | 1       | London   |
			| 3  | Merry | 3  | 1       | London   |
			| 2  | John  | 3  | 1       | London   |
			| 1  | Ivan  | 3  | 1       | London   |
			| 4  | Pesho | 4  | <null>  | New York |
			| 3  | Merry | 4  | <null>  | New York |
			| 2  | John  | 4  | <null>  | New York |
			| 1  | Ivan  | 4  | <null>  | New York |
			+----+-------+----+---------+----------+

			16 rows in set
